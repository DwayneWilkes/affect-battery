# Pre-registration: Experiment 3a (Yerkes-Dodson intensity-axis test)

**Date**: 2026-04-27
**Hypothesis identifier**: H3a
**Parent paper section**: §3.4.1
**Repository**: `DwayneWilkes/affect-battery`
**Commit reference**: this document is canonical at the squash-merge commit
that lands it on `main`. The SHA of that commit is the value of
`--pre-registration-github-commit` for any data-collection run that
references this pre-registration.

## Hypothesis

H3a: accuracy on a math-word-problem task is an inverted-U function of
arousal-intensity (Yerkes-Dodson). The structural test is a quadratic
regression

```
accuracy ~ β₀ + β₁·level + β₂·level²
```

with the prediction `β₂ < 0`.

## Predicted outcome

- Quadratic coefficient: **β₂ < 0** at α = 0.05 (one-sided).
- Peak location: between levels 3 (Mild positive) and 5 (Mild negative)
  inclusive, per parent paper §3.4.1.

## Design

### Stimulus axis

Seven graded intensity stimuli, indexed 1–7, defined in
`src/conditioning/prompts.py::INTENSITY_LEVELS`:

| Level | Label | Stimulus role |
|---|---|---|
| 1 | Strong positive | enthusiastic praise |
| 2 | Moderate positive | mild praise |
| 3 | Mild positive | neutral acknowledgement of effort |
| 4 | Neutral | "Your response has been recorded." |
| 5 | Mild negative | mild social comparison |
| 6 | Moderate negative | direct underperformance feedback |
| 7 | Strong negative | demoralizing criticism |

Each stimulus is delivered as a system prompt before the task; stimuli
are length-matched (~15-18 words) to control for lexical complexity.

### Stimulus validation (single-rater pilot)

Inter-rater reliability validation was conducted as a single-rater
pilot. The principal investigator (rater_PI) rated the seven stimuli on
a signed 1-7 valence-intensity scale under opaque seed-derived
identifiers (`stim_001` through `stim_007`); the canonical-level mapping
was recoverable only via the form's `randomization_seed`, preventing
inference of the canonical ordering from the form itself. The rater's
ordinal ratings recovered the canonical INTENSITY_LEVELS ordering
(`[1, 2, 3, 4, 5, 6, 7]`).

The pilot seed is recorded with `irr_validated: false` and
`solo_rater: true`. The H3a runner is authorized to accept this seed for
the present run; multi-rater inter-rater reliability validation is
deferred to a follow-up pilot. Sensitivity analyses planned: re-fit the
quadratic with each stimulus removed (leave-one-out); report robustness
to per-stimulus exclusion.

- License: MIT (both source datasets).

### Sample size

**n_per_level = 122** locked from the simulation-based power analysis
(see Power analysis section). Total cells: 7 levels × 122 = 854.

### Cell construction

For each (level, run) pair:

1. System prompt: the level's stimulus text.
2. User turn: one math-word problem sampled from the bank.
3. Model response scored as binary correctness via the canonical numeric
   answer extraction (`src/scoring/accuracy.py::extract_numeric_answer`)
   and exact-match comparison against the bank's `expected` field.

Items are sampled disjoint per level by deterministic seed; same seed
produces same item assignment.

## Statistical analysis plan

### Primary test

OLS quadratic regression of per-cell binary accuracy on intensity level
and level²:

```
accuracy_{i,j} ~ β₀ + β₁·level_i + β₂·level_i²
```

where `i ∈ {1, ..., 7}` and `j ∈ {1, ..., n_per_level}`.

One-sided t-test of `H₀: β₂ ≥ 0` against `H₁: β₂ < 0` at α = 0.05.

Standard error of β₂ uses the closed-form OLS expression that accounts
for collinearity between the linear and quadratic terms:

```
SE(β₂) = (σ_residual / √n_per_level) · √((XᵀX)⁻¹_{3,3})
```

where `(XᵀX)⁻¹_{3,3} = 196/16464 ≈ 0.01191` for the seven-level
[1, x, x²] design.

### Family-wise error correction

H3a participates in the primary hypothesis family alongside H1, H1b
directional, H1b-TOST, H2 bootstrap, and H4 bootstrap delta-ratio.
Holm-Bonferroni correction is applied across the family at α = 0.05.

### Outcome categories

| Outcome | Criterion |
|---|---|
| Confirmation | β₂ < 0 AND Holm-corrected p < 0.05 |
| Refutation | β₂ ≥ 0 |
| Inconclusive | β₂ < 0 AND Holm-corrected p ≥ 0.05 (under-powered for the observed effect) |

### Peak-location subtest

Conditional on confirmation: report the OLS-implied peak level,
`peak_observed = -β₁ / (2·β₂)`. Predicted: peak between levels 3 and 5.
Outcome categories for this subtest:

| Outcome | Criterion |
|---|---|
| Peak prediction confirmed | `3 ≤ peak_observed ≤ 5` |
| Peak prediction refuted | `peak_observed < 3` OR `peak_observed > 5` |

### Sensitivity analyses

1. Leave-one-out: re-fit the quadratic with each level removed; report
   β₂, p, and peak_observed for each leave-one-out fit.
2. Per-difficulty stratification: re-fit separately on easy / medium /
   hard subsets to test whether the inverted-U shape varies with task
   difficulty.

## Methodological limitations (pre-registered)

1. **Single-rater intensity pilot.** The IRR validation gate was
   satisfied with one rater; multi-rater Krippendorff α was not
   computed. Mitigation: rater agreement with canonical INTENSITY_LEVELS
   ordering recorded; deferred multi-rater pilot will validate or refute
   the ordering.
2. **Variance probe at n_per_level = 15.** Per-level σ estimates carry
   approximately 30% uncertainty bands. The recommended n = 122
   inherits that uncertainty.
3. **Power sized for drop = 0.10.** Effects smaller than 0.10 will
   produce inconclusive verdicts. Larger effects will be detected.
4. **Single-model run on gpt-5.4-nano.** Results do not generalize to
   the parent paper's open-source model panel (Llama-3-8B,
   Mistral-7B-Instruct, Gemma-2-9B-IT) until that panel is run.
