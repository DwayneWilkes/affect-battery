## What changed

A scoring-extractor bug in `src.scoring.accuracy.extract_numeric_answer` caused systematic undercounting of correct answers in the H3a corpora. The extractor's equals-sign priority tier used `re.search` (first-match) on the model response; chain-of-thought responses with intermediate calculations (e.g. `6 × 75 = 450 ... Total = 770`) were scored against the first match (450) instead of the final answer (770). The fix at commit f175e8d changes each priority tier to last-match.

Re-scoring the parent corpus reveals 266 of 854 cells (31%) flipped from incorrect to correct, with only 3 flipping the other direction. Re-scoring the within-subjects corpus flipped 238 of 854 cells (28%) from incorrect to correct, with zero flipping back.

The corrected verdict on the parent pre-registration's primary hypothesis changes from REFUTATION to INCONCLUSIVE.

## Corrected verdict: INCONCLUSIVE

Per the parent pre-registration's outcome categories:

| Outcome | Criterion |
|---|---|
| Confirmation | β₂ < 0 AND Holm-corrected p < 0.05 |
| Refutation | β₂ ≥ 0 |
| **Inconclusive** | **β₂ < 0 AND Holm-corrected p ≥ 0.05 (under-powered for the observed effect)** |

Under correct scoring, β₂ is negative across all five fits in the sensitivity analysis chain. None reach Holm-corrected significance at α = 0.05 within the primary hypothesis family. The hypothesis is supported in direction but not in magnitude at the pre-registered n.

## Rationale

The scoring bug is a code defect, not a methodology choice. The parent pre-registration's design specifies "binary correctness via the canonical numeric answer extraction (`src/scoring/accuracy.py::extract_numeric_answer`) and exact-match comparison against the bank's `expected` field." The bug caused the extractor to deviate from its canonical contract on chain-of-thought responses; correcting the bug restores the contract.

The amendment is filed under the parent pre-registration's `## Amendments` clause: "Any deviation from this pre-registration triggers a new commit on `main` recording the deviation rationale and a new git tag." The deviation here is between intent and behavior; the amendment is the public record of the correction.
