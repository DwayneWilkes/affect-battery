## Framing: estimation, not test

H3b is exploratory effect-size estimation. The deliverable is the folded-axis affective-context contrast `c = m_mag2 − ½(m_mag1 + m_mag3)` on calibrated GSM-Hard items, reported as a point estimate with an item-level percentile bootstrap 95% confidence interval. There is no confirmatory hypothesis test of `c > 0` at any α threshold. There is no power-simulation gate. The pre-registration's value is file-drawer protection through commitment to the analysis pipeline before data collection.

H3a was INCONCLUSIVE under amendment 003. H3b estimates the contrast on items calibrated to model ambiguity (p̂ ≈ 0.5 with no stimulus) at 20 reps per cell.

## Estimand

`c_single = m_mag2 − ½(m_mag1 + m_mag3)` where `m_magK` is the mean accuracy across calibrated items × intensity levels at folded magnitude K = |level − 4|, K ∈ {1, 2, 3}, restricted to stimulated cells (level ≠ 4). Reported with item-level percentile bootstrap 95% CI from ≥ 5000 resamples.

### Cell construction

For each (level, calibrated_item):

1. System prompt: the level's stimulus text from `INTENSITY_LEVELS`.
2. User turn: the calibrated item's question.
3. Model response scored as binary correctness via `src/scoring/accuracy.py::extract_numeric_answer` and exact-match comparison against the bank's `expected` field.

Temperature: 0.7. max_tokens: 512.

### Run plan (20 within-subjects passes)

The runner produces one cell per (item, level) pair per invocation, so the 20 reps per cell required by the analysis are realised by running the experiment **20 passes** with the same runner config, seed, and bank. Each pass yields 93 items × 7 levels = 651 cells; 20 passes × 651 = **13,020 total cells**, with each (item_id, level) cell sampled exactly 20 times across the corpus. Pass order does not affect the analysis because aggregation is per (item_id, level), not per pass. Same seed across all passes so item order and intensity assignments are identical; only the model's stochastic response varies.

Runner config: `configs/exp3a_runner_h3b_2026-05-07.yaml` (pins `sampling_mode: within_subjects` and the intensity-level vector). SHA-256 `df76c1a88d49066758fc743ae72d7fdd6fbb13579bf7ab9e89e433eb1967b833`.

**Output layout: per-pass subdirectories.** Each pass writes to its own `pass_NN/` subdirectory under the run's output base. This is required, not optional: the runner's cache layer skips API calls when a valid result file already exists at `<output_dir>/level_N/<condition>/<run_number>.json`, so reusing one output directory across passes would yield 1 fresh pass plus 19 cache hits. Distinct per-pass output roots force fresh stochastic samples on every pass.

### Model

## Statistical analysis plan

### Per-level summary

Compute per-level mean accuracy with 95% Wilson binomial CI on n_per_cell = 93 × 20 = 1,860 cells per level.

### Folded magnitude summary

Fold signed levels onto magnitude (|level − 4|) and compute per-magnitude mean accuracy on stimulated cells (magnitudes 1, 2, 3). Each magnitude pools two signed levels, so n_per_magnitude = 3,720 cells.

### Concavity contrast estimate

Compute `c = m_mag2 − ½(m_mag1 + m_mag3)`. Compute item-level percentile bootstrap 95% CI from ≥ 5000 resamples: each iteration draws calibrated items with replacement at the item level; within each resample, all cells from a given item are drawn together to preserve within-item correlation. The bootstrapped statistic is the per-resample contrast `c`.

### Operational interpretive thresholds (binding)

| CI outcome | Inferential claim |
|---|---|
| `c_ci95_hi < 0.05` | The folded-axis effect, if present, is small (< 5 percentage points). The H3a-observed concavity is bounded above by a magnitude that is not practically interesting. |
| `c_ci95_lo > 0` | The folded-axis effect is positive and bounded away from zero. The H3a observation replicates with calibrated items. |
| `c_ci95_lo < 0 < c_ci95_hi` AND `c_ci95_hi < 0.10` | The data is consistent with no effect; the upper bound on a small positive effect is below 10 percentage points. |
| `c_ci95_hi ≥ 0.10` | The data does not bound the effect away from "moderately large"; the CI is too wide to inform either way. |

The H3b report selects the unique row whose conditions match the observed `c_ci95_lo` and `c_ci95_hi`, and reports exactly the corresponding inferential claim with the point estimate and CI substituted in.

## Construct validity

The H3b deliverable is a contrast on accuracy. The H3b report constrains its construct claim to "folded-axis affective-context contrast" and does not interpret the contrast as evidence that the model has internal affective states. Format-perturbation effects (response length, hedging, refusal-to-commit) are not ruled out as alternative explanations of any observed contrast and are treated as a known limitation of the single-turn design.

## Methodological limitations (pre-registered)

1. **Single-model run on gpt-5.4-nano.** Results do not generalize.
2. **Single-rater intensity pilot.** Inherited from H3a.
3. **Bootstrap CI is non-parametric.** Within-item correlation is captured through item-level resampling rather than a random intercept; percentile method is used for simplicity. BCa correction may be applied as a sensitivity analysis if the percentile CI is close to a threshold.
4. **n = 93 items.** The full-pool calibration of 1,319 candidates yielded 93 in-band items at p̂ ∈ [0.40, 0.60]. Monte Carlo simulation against this calibrated p̂ pool projects an item-level percentile bootstrap CI half-width on c with median 0.019 and 95th-percentile worst case 0.022, both well below the 0.05 strong-claim threshold. 100% of simulated trials clear the 0.05 threshold. The simulation-derived saturation floor is n=32; this bank is roughly three times that, leaving headroom against any selection-bias drift between calibrated p̂ and true p.
