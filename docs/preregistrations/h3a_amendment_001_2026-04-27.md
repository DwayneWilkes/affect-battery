## What did not change

- Hypothesis: β₂ < 0 in the OLS quadratic regression of accuracy on
  intensity level.
- Predicted peak: between levels 3 (Mild positive) and 5 (Mild negative)
  inclusive.
- Sample size: `n_per_level = 122`, total cells `7 × 122 = 854`.
- Stimulus axis: `INTENSITY_LEVELS` in `src/conditioning/prompts.py`.
- Cell construction: single-turn paradigm, system message =
  level's `feedback_text`, user message = one bank item.
- Statistical analysis plan: one-sided t-test on β₂ at α = 0.05 with
  Holm-Bonferroni correction across the primary hypothesis family.
- Outcome categories: confirmation / refutation / inconclusive (with the
  conditional peak-location subtest).
- Methodological limitations: single-rater pilot, σ uncertainty at
  n_per_level = 15, drop = 0.10 minimum effect, single-model run on
  gpt-5.4-nano.
- Sensitivity analyses: leave-one-out per level, per-difficulty
  stratification.

## Rationale

The parent pre-registration commits to disjoint-per-level item
sampling: each level's `n_per_level` items are drawn from the bank
without sharing items across levels. With `n_per_level = 122` and
`n_levels = 7`, this requires the bank to hold at least
`7 × 122 = 854` items. The original 200-item bank did not satisfy
this constraint at the pre-registered `n_per_level`.

Two alternative remedies were considered and rejected:

- **Reducing `n_per_level` to 28** (the maximum compatible with
  cross-level disjoint sampling on a 200-item bank). Rejected because
  the parent pre-registration's power analysis is sized for the
  smallest detectable effect (drop = 0.10 from peak to edges) at
  `n_per_level = 122`. Reducing to 28 drops below the recommended n
  for 80% power on that effect size.
- **Relaxing the disjointness contract to within-level only**.
  Rejected because cross-level disjoint sampling is the
  methodologically meaningful guarantee: it ensures that per-level
  accuracy estimates are not artificially correlated by item identity.
  Relaxing the contract would introduce a between-level item-sharing
  confound at the smallest scale where the contract bites.

Bank expansion preserves both the pre-registered `n_per_level` and the
disjointness contract.

## Source preservation

The amendment uses the same sourcing script and the same seed (42) as
the original bank. The script's stratified-sampling logic is
deterministic: re-running with `--total-items 854` produces the bank
recorded under the new SHA without any ad-hoc selection.

The original 200-item bank's SHA
(`ff6945f4b78c788c0d6cf77516fc6a9d35fc1d8ca8f928d6c68d6564c864cb93`) is
preserved in the parent pre-registration commit's
`configs/banks/gsm8k_v1.yaml`. Anyone re-running against the parent
pre-registration's `--pre-registration-github-commit` reference will
read the original 200-item bank; runs against this amendment's commit
read the 854-item bank.

## Variance-estimate carryover

The original variance probe at `n_per_level = 15` measured per-level
σ in the Bernoulli ceiling range (0.26 to 0.52, mean accuracy 0.07 to
0.47). At those mean accuracies, σ ≈ √(p(1 - p)) is bounded in
[0.30, 0.50] across the new bank's expected accuracy range. The power
analysis therefore remains valid for the amended bank.

A confirmatory variance probe at `n_per_level = 15` against the
amended bank can be conducted as a sensitivity check. The `n = 122`
production run's per-level σ values are also recorded post-hoc and
participate in any post-hoc reproducibility audit.

bank path, and the bank SHA-256 in every result manifest. Comparing
the recorded `bank_sha256` against the parent pre-registration's
declared SHA versus the amendment's SHA is the mechanical check that
identifies which pre-registration vehicle the run is operating under.
